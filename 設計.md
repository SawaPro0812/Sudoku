あなたはReactとTailwind CSSを熟知した、優秀なフロントエンドエンジニアです。
以下の要件に従って、iPhoneの画面サイズと操作性に完全に特化した「究極の数独（Sudoku）ウェブアプリ」を作成してください。

# 技術スタック
・Vite + React (TypeScript)
・Tailwind CSS v4
・lucide-react（アイコン）
・PWA: manifest.json + Service Worker

# 1. ゲームの基本機能とロジック
・盤面自動生成：バックトラッキング等のアルゴリズムを使用し、解が1つになる正しい数独盤面を生成する。
・難易度：初級（ヒント38個）、中級（ヒント30個）、上級（ヒント24個）の3段階。
・入力モード切替：通常入力モードと、1マスに複数候補を記録できるメモ（鉛筆）モード。
・回答チェック：現在の入力に矛盾がないか判定し、エラー箇所を赤くハイライトする。クリア時にはお祝いのポップアップを表示する。
・ミス回数カウント：入力時に正解（solution）と比較し、間違いの回数をカウントしてヘッダーに表示する。
・ヒント使用回数カウント：ヒントボタンの使用回数をカウントしてヘッダーに表示する。

# 2. 追加の高度な機能
・タイム計測：ゲーム開始時からの経過時間を「MM:SS」形式で画面上部にリアルタイム表示する。
・オートセーブ（途中保存）：
　`localStorage` を使用し、盤面の状態、現在のタイム、選択中の難易度、メモの状態、ミス回数、ヒント回数を自動保存する。
　次回ブラウザ（またはアプリ）を開いた際に、前回の続きから再開できるようにする。
・ヒント機能：
　「ヒント」ボタンを押すと、まだ埋まっていないマスのうち1箇所に、正しい数字を自動で入力して確定させる。
・クリア時ポップアップ：
　クリア時に紙吹雪アニメーション付きのモーダルを表示し、タイム・難易度・ミス回数・ヒント回数を一覧で表示する。

# 3. iPhone特化のUI/UX要件（最重要）
・モバイルファーストデザイン：PCからの閲覧時も、画面中央にiPhoneサイズのコンテナ（max-w-md）を配置し、スマホアプリのような見た目にする。
・iPhoneノッチ / Dynamic Island 対応：
　- `env(safe-area-inset-top)` を使用してカメラ部分を避けるヘッダーを配置する。
　- ヘッダーは背景色付きで、タイトル・難易度・新規ボタン・統計情報（タイマー、ミス回数、ヒント回数）を表示する。
・盤面の枠線：
　- 9x9グリッドはCSS gridで描画し、各セルに細い罫線（0.5px）を付ける。
　- 3x3ブロック境界には太い罫線（2px）を確実に表示する（セル右辺・下辺に`border-right-thick`/`border-bottom-thick`クラスを適用）。
・タッチ操作への最適化：
　- 誤タップを防ぐため、数字ボタンや操作ボタン（消去、ヒント、メモ切替）は画面下部に大きく配置（ボトムナビゲーション風）する。
　- 数字ボタンは5個＋4個の2行レイアウトとし、高さ56px・フォントサイズ2xlで十分な大きさを確保する。
　- 盤面のマス（9x9）は、iPhoneの画面幅にいっぱいに広がるようにしつつ、タップしやすいサイズを確保する。
　- タップ時のハイライトや、同じ数字が盤面にある場合の強調表示を実装し、視認性を高める。
・ダブルタップズームの防止：CSSで `touch-action: manipulation` を全要素に適用し、ボタン連打時に画面がズームされないようにする。
・アイコン：UIのボタンには `lucide-react` のアイコンを使用し、直感的に操作できるようにする。

# 4. PWA対応
・`manifest.json`：アプリ名「数独」、standalone表示、portrait固定、テーマカラー設定。
・Service Worker（`sw.js`）：キャッシュファースト戦略でオフライン対応。
・`index.html`：viewport設定（`maximum-scale=1.0, user-scalable=no`）、apple-mobile-web-app関連meta、manifest linkを記載。
